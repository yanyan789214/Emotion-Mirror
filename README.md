## 这是什么

**情绪镜像**是一个小型 Web 应用。使用者在网页上写下自己近期真实、复杂、
说不清的处境(比如一次抉择、一段关系、一种说不出口的情绪),点击"开始照见"后,
后台会调用 AI(通义千问 Qwen),从 5 个维度分别找出 3-5 个曾经历过相似处境的
"回声":

| 维度 | 说明 |
|---|---|
| 🎬 电影 Movie | 真实存在的电影里,某个角色面临过相似处境 |
| ✍️ 诗歌 Poem | 意境与处境有共鸣的真实诗作 |
| 📜 历史 History | 真实历史人物经历过的相似处境及后续转变 |
| 📖 小说 Novel | 真实小说里的情节或角色心理转变 |
| 💭 其他 Other | 哲人名言 / 画作 / 音乐等其他形式的联想 |

每一条"回声"可以点开看完整的起因经过结果,还能针对这一条继续追问细节
(比如"具体发生了什么""后来他怎么样了")。界面支持中文 / English 切换。

项目采用**前后端分离**结构:浏览器端的网页不包含任何密钥,所有对 AI 的调用
都经过你自己的后端服务器代理,密钥只保存在服务器的环境变量里。

---

## 功能一览

- ✅ 输入处境 → 五维度、每维度 3-5 条案例
- ✅ 案例详情页 + 针对单条案例的多轮追问对话
- ✅ 中文 / English 双语界面(含 AI 生成内容的语言切换)
- ✅ 加载中的进度条 + 阶段提示文案 + 计时,避免"卡住了"的困惑
- ✅ 详情页附带"去搜索引擎查证"的链接
- ✅ 后端限流(防止接口被刷、控制 API 调用成本)、输入长度校验
- ✅ AI 密钥只存在于后端 `.env`,浏览器代码和网络请求里都看不到

---

## 技术栈

- **前端**:原生 HTML / CSS / JavaScript(无需构建工具,双击打不开是因为要经过后端,见下方"快速开始")
- **后端**:Node.js + Express
- **AI 服务**:通义千问 Qwen(通过 OpenAI 兼容接口调用)

## 快速开始

### 前置条件

- 已安装 [Node.js](https://nodejs.org/)(建议 18 版本或以上)
- 一个可用的通义千问(Qwen)API 密钥

### 步骤

```bash
# 1. 解压项目后,进入后端目录
cd emotion-mirror-v4/backend

# 2. 复制环境变量模板
cp .env.example .env

# 3. 用编辑器打开 .env,填入你的真实密钥
#    QWEN_API_KEY=sk-xxxxxxxxxxxxxxxx

# 4. 安装依赖
npm install

# 5. 启动服务
npm start
```

看到终端输出 `情绪镜像 v4 后端已启动: http://localhost:3000`,说明启动成功。

打开浏览器访问 **http://localhost:3000** 即可看到网页并直接使用
(后端已经顺便把 `frontend` 目录当静态网站托管了,不需要再单独起一个前端服务)。

> 💡 只有你自己电脑能访问这个地址。想让别人也能用,请看下方"部署到公网"。

## 环境变量说明

在 `backend/.env` 里配置(先 `cp .env.example .env` 再编辑):

| 变量名 | 必填 | 说明 |
|---|---|---|
| `QWEN_API_KEY` | ✅ | 你的通义千问 API 密钥,**不要提交到 git、不要分享给别人** |
| `QWEN_BASE_URL` | 否 | Qwen 的 OpenAI 兼容接口地址,默认使用阿里云百炼 DashScope 的 `https://dashscope.aliyuncs.com/compatible-mode/v1`;如果你用的是其他网关/代理商,改成对应地址 |
| `QWEN_MODEL` | 否 | 模型名,默认为 `qwen3.6-flash` |
| `PORT` | 否 | 服务监听端口,默认 `3000` |
| `CORS_ORIGIN` | 否 | 允许跨域调用接口的前端来源,多个用逗号分隔;前后端同源部署(本文档默认方式)时可以不管 |

## 接口说明

后端一共暴露两个接口,前端 `app.js` 会自动调用它们,你一般不需要手动调用,
但如果想二次开发或调试,可以参考:

### `POST /api/mirrors` —— 生成五维回响

请求体:
```json
{ "situation": "用户写下的处境文字...", "language": "zh" }
```
- `situation`:必填,4-4000 字
- `language`:可选,`"zh"` 或 `"en"`,默认 `"zh"`

返回:
```json
{
  "movie": [ { "title": "...", "teaser": "...", "detail": "..." }, ... ],
  "poem": [ ... ],
  "history": [ ... ],
  "novel": [ ... ],
  "other": [ ... ]
}
```

### `POST /api/followup` —— 针对某条案例继续追问

请求体:
```json
{
  "entry": { "title": "案例标题", "detail": "案例详情" },
  "question": "用户追问的问题",
  "history": [ { "q": "上一轮问题", "a": "上一轮回答" } ],
  "language": "zh"
}
```

返回:
```json
{ "answer": "AI 针对这个问题的回答" }
```

两个接口出错时都会返回 `{ "error": "友好的错误提示" }`,详细的报错原因只会
打印在后端的终端日志里,不会泄露给浏览器。

## 常见问题排查

| 现象 | 可能原因 |
|---|---|
| 终端提示 `服务器未配置 QWEN_API_KEY` | `.env` 里没填密钥,或者忘了 `cp .env.example .env` |
| 点击"开始照见"后一直报错 | 检查 `QWEN_API_KEY` 是否正确、`QWEN_BASE_URL` 是否是你实际能访问的地址、服务器网络是否能访问该地址 |
| 生成的内容偶尔缺了某个维度 | AI 偶尔没有严格按格式输出,后端已经做了兜底(显示"解析异常,请重新生成"),重新点一次"开始照见"通常就好 |
| 页面打不开 / 显示"无法访问此网站" | 确认 `npm start` 还在运行、终端没有报错、访问的地址和端口跟终端里显示的一致 |

## 部署到公网

本地运行只有你自己能访问。如果想让其他人也能用:

1. **后端**:部署到任意支持 Node.js 18+ 的平台(Railway、Render、Fly.io,
   或自己的云服务器 + PM2 / Docker 均可),把 `QWEN_API_KEY` 等配置成
   该平台的环境变量。
2. **前端**:
   - 最简单的方式:让后端继续顺带托管(`server.js` 已经配置好了 `express.static`),
     部署好后端后,访问后端的公网地址即可看到完整网站。
   - 或者:把 `frontend` 单独部署到 Vercel / Netlify / Cloudflare Pages 等静态托管平台,
     此时需要在 `frontend/index.html` 的 `<script src="app.js">` 之前加一行:
     ```html
     <script>window.EMOTION_MIRROR_API_BASE = "https://你的后端域名";</script>
     ```
3. 生产环境建议把 `.env` 里的 `CORS_ORIGIN` 收紧为你真实的前端域名,而不是留空。

## 关于内容准确性

这些"回声"是 AI 结合公开知识生成的,**不是从权威数据库逐条核实过的确凿史实**。
Prompt 里已经要求 AI 尽量避免编造具体细节、简化因果关系,不确定的地方要
明确说"这一点历史记载有限",但仍然可能存在偏差。详情页附带的"去搜索引擎
查证"链接,建议在引用到正式场合前自行核实关键信息;"继续追问"得到的回答
同样是 AI 生成的延伸,不是替你查了原始资料。
